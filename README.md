## 👋 Hello, I'm Amber

### ✨ I'm a Full Stack Developer. <br /><br />

I grew up surrounded by websites and technology. I'm a millennial. Baby me practiced with Netscape Navigator, but that was some time ago. I had some exposure to CSS via Dreamwidth, an open source site which hosts fannish communities. Dreamwidth CSS runs a gamut. The main pages have better splash pages now, but profiles are still old school CSS. Recently there was a quality poll about UX at Dreamwidth and I would say that I think improving profiles to more modern splash-page-like CSS might enhance UX. Light animations can go a long way for certain organizational pages. But I'm also really curious in knowing more about Dreamwidth's architecture now. It dates back to Perl.

I'm coming to think the root of a well-optimizing developer is genuine curiosity. I realized I felt genuine curiosity at the idea of forking Dreamwidth and checking out the codebase. I've had this reaction to a few open source startups and there are a few startups I've wished were open source. 

It's my goal to contribute to more open source this year that genuinely excites me, but if I don't contribute, I might at least investigate. There are some really lovely and elegant open source codebases out there. I think this ecosysystem is a good way to get a lot of exposure to dfferent developer approaches.
